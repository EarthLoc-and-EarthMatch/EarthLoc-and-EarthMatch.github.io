Automated Geolocalization Techniques for Astronaut Photography

Astronauts on the International Space Station take thousands of photos per month, which are used for disaster management and climate change studies. Before being used however, these photos need to be localized: this was typically done manually, in a task that NASA defines "monumentally important, but monumentally time-consuming job". This project provides an automatic localization pipeline, based on a large-scale image retrieval model (EarthLoc - CVPR 2024) and a coregistration method (EarthMatch - CVPRW 2024).

Complete pipeline

The retrieval step finds a number of predictions for a query (the astronaut photo to be localized). The search is performed over a large database covering a huge area (the visible area from the ISS is 2M square km). The matching step takes the query and a candidate, and decides if the two images overlap and, if they do, it also provides an accurate coregistration.

Dataset

In order to determine the positions of the queries using image retrieval, it is necessary to have a global database of images, with each image tagged with its corresponding location. We create such a database using a publicly available compilation of Sentinel-2 satellite imagery, which offers cloudless coverage of landmasses worldwide with resolutions of up to 15 meters per pixel.

Matching iterations

The procedure starts with matching the query $Q$ and the candidate $C_0$ to produce a first homography $H_0$. The candidate surroundings $C_s$ is generated and $H_0$ applied to produce $C_1$. $C_1$ is then matched with $Q$, producing $H_2$ which applied to $C_s$ yields $C_2$. This iterative process continues for a fixed number of iterations or until convergence. Note the number of inlier matches increases steadily for each iteration.
